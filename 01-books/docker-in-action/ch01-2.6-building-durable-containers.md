# 2.6 Building durable containers

Softwares fail in rare conditions, the important thing is restoring them.

A basic staratergy for recovering from temporary failures is automatically restarting.

Docker provides a few options for monitoring and restarting containers.

## Automatically restarting containers

Docker provides a restart policy. Using the `--restart` at container-create time

- Never restart (default)
- Restart when a failure is detected
- Predetermined time to restart when a failure is detected
- Always restart
